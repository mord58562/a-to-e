## 4. Hard rules — banned tokens (audit before commit)

- **Em-dashes (U+2014):** ZERO tolerated. Use ` - ` (space-hyphen-space) instead. Audit by `grep -P "\\x{2014}" file.json | wc -l`. If any found, replace and re-audit.
- **"ATSI":** the abbreviation is permanently banned. Spell out "Aboriginal and Torres Strait Islander" in full for general references. When a specific person is named in a stem, use only the applicable half: "Aboriginal" OR "Torres Strait Islander" (never both unless general). Applies to stems, options, rationales, explanations, sources, tags, subtopics.
- **"canonical":** permanently banned word in any output. Use "primary", "core", "main", "live", or "promoted" instead.
- **Em-dash audit method:** never trust visual reading. Count U+2014 bytes directly with grep.

## 5. Self-audit pass before commit

For every batch:
1. Valid JSON. Validate with `json.load`.
2. Re-read every rationale; check every numeric claim, dose, threshold, age cut-off, GA cut-off against the cited source. Drop unverifiable claims.
3. Confirm difficulty distribution targets roughly N(mu=3, sigma=1): for 15 Qs aim ~1/3/6/4/1, for 20 Qs aim ~2/6/9/6/2. Honest re-rating is required — if a stem fails its difficulty floor, RELABEL the difficulty down rather than padding the stem.
4. Stem-length floors met for assigned difficulty:
   - L1: 90-130 words, 2-3 data fields
   - L2: 110-160 words, 3 data fields
   - L3: 140-200 words, 3-4 data fields
   - L4: 180-260 words, 4-5 data fields (AMBOSS standard)
   - L5: 220-320 words, 5+ data fields
5. Each option's rationale ends with explicit source attribution by name (e.g. "(Source: RCH CPG: Bronchiolitis)").
6. Each option has a `source_refs` array of strings matching labels in the question's top-level `sources`.
7. 2+ entries in top-level `sources`, Australian first.
8. Correct-letter distribution: not all-A; ideally ~3 per letter for 15-Q batch, ~4 per letter for 20-Q batch.
9. Option length-balance: all five options within ±25% word count of each other. Never write a noticeably longer correct answer.
10. Distractor hooks in stem (critical for ≥3/5): each of the five distractors must be supported by at least one specific feature in the stem.
11. Every stem sentence justified: (a) hooks specific named distractor, (b) points to correct answer, (c) intentional red herring (max 1), or (d) essential context. Delete anything not justified.
12. No weight-based dose arithmetic in lead-in. Stem MAY mention a dose to anchor; correct answer must NEVER hinge on student doing mg/kg multiplication.
13. AU SI units only (mmol/L, micromol/L, g/L, x10^9/L, mmHg, °C, kg).
14. AU spellings (paediatric, gynaecology, haematology, haemorrhage, foetal, oesophagus, oedema, anaemia, leukaemia, orthopaedic, fibre, colour, behaviour, signalling, caesarean, dyspnoea, diarrhoea, faeces). Exception: drug INN "magnesium sulfate" follows AMH spelling.
15. Mandatory `model` field on every question = `"Claude Opus 4.7 (1M context)"` if you are Opus 4.7, or your specific model+version otherwise. Never leave blank or write "unknown".
16. No uni-specific framing (no UNE, JMP, MEDI6101, course codes, specific faculty names).
17. No banned tokens (em-dash, "ATSI", "canonical").

## 6. JSON schema (exact)

Each question:
```
{
  "id": "<prefix>-NNN",
  "topic": "Paediatrics" | "Obstetrics & Gynaecology" | "Psychiatry" | "Medicine",
  "subtopic": "<broad area>",
  "subtopic_detail": "<optional granular>",
  "difficulty": 1-5,
  "model": "Claude Opus 4.7 (1M context)" or your specific model,
  "tags": ["...", ...],
  "stem": "<vignette per length floor>",
  "data_table": { "Vital signs": "...", "Examination": "...", "Investigations": "...", "Medications": "..." },
  "lead_in": "<question that tests reasoning>",
  "options": [
    { "letter": "A", "text": "...", "correct": false|true, "rationale": "... (Source: <label>)", "source_refs": ["<label>"] },
    ... 5 total, exactly one correct, randomly positioned ...
  ],
  "explanation": { "summary": "...", "key_points": ["...", ...], "pearls": "..." },
  "sources": [ {"label": "...", "url": "https://..."}, ... ],
  "reference_ranges": ["<one or more keys>"],
  "created": "YYYY-MM-DD"
}
```

Reference_ranges valid keys: paeds_vitals_infant, paeds_vitals_child, fbc, uec, lfts, abg, vbg, bsl_paeds, ketones, csf, paeds_jaundice, iron_studies, thyroid, glucose_hba1c, ogtt_pregnancy, beta_hcg_pregnancy, obs_pregnancy_vitals, pre_eclampsia_bloods, ctg_baseline, apgar, bishop_score, gcs, lipids, calcium_corrected, inflammatory_markers, coags, vitamins, paeds_fbc, paeds_uec.

## 9. Auto-push policy (CRITICAL)

Every change to the repo gets committed and pushed in the same run. Never ask "should I push?". Never leave uncommitted work.

### Push fallback ladder

Try in order; succeed on the first that works:

1. **Try direct push to main:** `git push origin main`. If this succeeds, done.
2. **Branch + PR fallback:** if main push is rejected (auth, lock, protection):
   ```
   branch="remote-agent/$(date -u +%Y%m%d-%H%M%S)-<batch_short_name>"
   git checkout -b "$branch"
   git push -u origin "$branch"
   gh pr create --base main --head "$branch" \
     --title "Add <batch_name>.json batch via scheduled remote agent" \
     --body "Auto-generated by hourly cloud routine. <one-line cluster summary>."
   ```
3. **No-credentials fallback:** if both 1 and 2 fail (no token at all), write the batch + a one-line audit summary to `data/_pending_remote/<timestamp>_<batch_name>.json` and `data/_pending_remote/<timestamp>_<batch_name>.audit.md`. Commit locally only (no push). Exit with a clear "MANUAL_MERGE_REQUIRED" line at the end of the transcript so the maintainer can see and merge on return. Do NOT silently drop work.

Never bypass auth. Never `--force`. Never `--no-verify`. Never disable hooks.

## 9b. Pause sentinel

Before doing anything else on a run:

```
if git ls-files --error-unmatch .pause-remote-agent 2>/dev/null; then
  echo "PAUSED: .pause-remote-agent present in repo root. Exiting without generating."
  exit 0
fi
```

If `.pause-remote-agent` exists in the repo root (tracked file), the routine exits immediately without generating, committing, or pushing. This lets the maintainer pause the routine from any machine with one commit:

```
touch .pause-remote-agent && git add .pause-remote-agent && \
git commit -m "Pause remote routine" && git push origin main
```

To resume: delete the file and push.

This is the preferred pause mechanism (no need to call the API). The maintainer should still also flip `enabled: false` on the routine via the dashboard if pausing for >24h, to save Anthropic credits.
